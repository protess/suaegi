# Plan — git-depth read 레이어 (`suaegi-git` 확장) 확정

조사: `docs/superpowers/research/2026-07-24-git-depth.md` (Orca @ v1.4.150-rc.0,
인용 file:line 고정). Codex 교차검증 판정 **AFTER-FIXES** 반영(모든 op 분류 확인,
파서 재사용성·format 오프셋·인코딩 등 4수정 + 8질문 확정). 이 문서가 구현 계약이다.

## 0. 결정 (조사 + Codex 확정)

coding-agent-on-worktrees UI가 필요로 하는 **read-mostly 자율-검증 git 연산**. 전부 GitRunner
경유(네트워크 0, 사용자 config 미접촉, 데이터손실 0). **stash/blame/cherry-pick/revert/reset은
Orca git에 없음**(ground truth 0) → skip. push/pull/fetch는 네트워크+creds라 별건 defer
(suaegi-secrets 선결). checkout/ahead-behind counts/graph swimlanes도 defer(§3).

## 1. Codex 반영 픽스 (구현자 필독)

- **F1 — compare.rs 파서 추출 선결 (M0).** `compare.rs:236-267`(numstat `-z`)·`:269-319`
  (name-status `-z`)가 `branch_compare` 본문 인라인이라 재사용 불가. **먼저 추출**:
  `pub(crate) fn parse_numstat_z(...)`, `pub(crate) fn parse_name_status_z(...)`. `branch_compare`가
  이걸 호출(순수 리팩터, 기존 테스트 무변경 통과=행동 no-op). `file_diff`/`file_head_bytes`
  (`:495,:533`)는 이미 `pub async fn`이라 그대로 재사용.
- **F2 — history `--format` byte-for-byte(죽은 `%ct` 포함).** Orca(`git-history-log-parser.ts:5-6`):
  `%H%n%aN%n%aE%n%at%n%ct%n%P%n%(decorate:prefix=,suffix=,separator=%x1f)%n%B`. 7줄 헤더
  `[0]=H [1]=aN [2]=aE [3]=at [4]=ct(미사용) [5]=P [6]=decorate`, `[7..]=%B`. **`%ct`를 "정리"하면
  하위 인덱스 shift로 parents/decorations/body 손상** — 그대로 이식하거나 의도적 renumber+테스트.
- **F3 — decoration은 `%x1f`-split만, git ≥2.34 요구.** Orca `parseGitDecorationRefs`
  (`git-history.ts:17-76`)의 `includes(SEP)?split(SEP):split(',')` comma 폴백은 git <2.34용 —
  suaegi는 modern git 가정(compare.rs "실측 git 2.50.1")이라 **`%x1f`-split만 이식, comma 폴백 skip**,
  플랜에 git ≥2.34 전제 명시.
- **F4 — NUL-split 후 레코드별 leading `\n` strip.** `parseGitHistoryLog`(`git-history.ts:101-102`):
  `stdout.split('\0')` **후** 각 레코드 `replace(/^\n+/, '')` — `-z`에서 각 레코드가 git pretty-print의
  leading `\n`을 달고 와서 안 벗기면 line[0](hash)이 빈값→hash 정규식이 커밋 통째 drop. **≥2 커밋
  fixture로 레코드 수 단언**(mutation-target).

## 2. 마일스톤

### M1 — Log/history (`history.rs`, 신규) — keystone
워크트리 커밋 목록 = "에이전트가 한 일". `git log --format=<F2 그대로> -z --topo-order
--decorate=full -n<limit+1> HEAD` 파싱 → commits + decorations. **transient≠false-negative**:
unborn HEAD(fresh repo)·detached HEAD → **empty/fallback, 절대 에러 아님**(Orca `resolveCommit`
null→`{items:[], hasIncoming:false, hasOutgoing:false, hasMore:false, limit}` `git-history.ts:164-173`;
detached는 `symbolic-ref` 실패→`{id, name:shortHash, category:'commits'}` fallback `:85-112`).
limit clamp `[1,200]`(default 50, `git-history-types.ts:23-24`), **hasMore = `-n limit+1` fetch 후
`parsed.len > limit`, items = 앞 `limit`**. hasIncoming/hasOutgoing 불린은 merge-base call에 무료로
탑승(§3 ahead/behind counts는 defer). 선택: `dedupeRemoteTrackingRefs`(`git-history-ref-display.ts:11`,
동일 커밋의 `origin/x`+로컬 `x` 중복 pill 제거, cosmetic) + decoration category 정렬.
*crux/mutation:* 잘못된 separator, `%ct` drop 오프셋 shift, leading-`\n` strip 누락, un-clamp limit,
unborn/detached가 에러 반환. 인코딩은 lossy `run()`(기존 convention, author/body 표시-전용).

### M2 — Commit-show (`commit_show.rs`, 신규 — M0 추출 후)
단일 커밋 diff, 파일별. **M0 추출 파서(`parse_numstat_z`/`parse_name_status_z`) + `file_diff`/
`file_head_bytes` 재사용.** root commit(부모 없음)→`diff-tree --root --no-commit-id`(하드코딩 empty-tree
해시 **아님**, `git-handler-commit-diff-ops.ts:70-104`); first-parent via `rev-list --parents -n1` +
2번째 whitespace 필드(root면 1필드→null→`--root` 분기 트리거). **argv-injection 가드**: commit ref는
git 호출 전 **정확히 40-or-64 hex**(exact-alternation, `{40,64}` 범위 아님 — 범위는 41..63도 통과)
검증. `refname.rs:5-13 validate_user_ref` 패턴, **mutation 검증 필수**. **`-z` numstat 사용, Orca text-form
brace `{old => new}` 언랩 미이식**(`-z` 두-레코드 rename이 그 버그 클래스 회피).
*crux/mutation:* argv 가드(비-hex/leading-`-`/`HEAD~1` 거부), root-commit `--root`, rename `-z` 두-경로.

### M3 — Branch read (`branch.rs`, 신규) — list-only
`for-each-ref` 목록 + current(`checkout.ts:41`), 피커용. checkout은 **defer**(소비자 없음, worktree-per-branch
모델이라 in-place 스위치 희소). trivial/pure.

## 3. Deferred (명시)
push/pull/fetch(+순수 헬퍼 `stripCredentialsFromMessage` — 실 flow 통합 커버리지 위해 remote 마일스톤과
함께), branch rename/create/delete(worktree 수명과 융합, 일부 worktree.rs 소유), ahead/behind **counts**
(cherry-mark probe 비자명, M1 불린으로 충분), graph swimlanes(render view-model, UI), commit checkout.

## 4. 순서 (확정)
M0(compare.rs 파서 추출, M2 첫 PR로 흡수) → M1 history → M2 commit-show → M3 branch read.
