# git-depth — porting Orca's remaining git operations to suaegi-git

Research doc. **Not an implementation.** Scopes the next `suaegi-git` backend milestone
*beyond* the write-ops (stage/commit/discard) already landed. Orca ground truth pinned at
`orca-src` @ v1.4.150-rc.0; every citation is `file:line` under that tree.

## What suaegi-git already has (do NOT re-scope)

worktree create/list/delete (`worktree.rs`), merge-base diff (`compare.rs`: `branch_compare`,
`file_diff`, `ChangedFile`/`ChangeStatus` incl. `Copy`, `-z` name-status **and** `-z` numstat
parsers with rename/copy two-path handling and `core.quotePath=false`), working-tree status +
ignore + conflict-kind (`status.rs`/`conflict.rs` — note `conflict.rs:24,100` already models a
`CherryPick`/`CHERRY_PICK_HEAD` conflict state), path-safety (`resolve_preserve_symlink`,
`ExistingOnly`), safe fs (`fs.rs`), **staging + commit + discard write-ops** (`write_ops.rs`,
incl. the `DiscardOutcome` data-loss discipline), Quick Open (`quick_open.rs`), the `GitRunner`
(`run`/`run_with_timeout`/`run_expecting`/`run_bytes`/`run_with_stdin`, cap/timeout/tree-kill/
`GIT_TERMINAL_PROMPT=0`, never injects identity or `-c` config overrides), and the
`git init`-in-tempdir fixture harness. `worktree.rs:129,209` already runs `branch -d`/`branch -D`
internally as part of worktree cleanup — there is no *standalone* branch op yet.

## §0 — Per-operation survey (Orca-cited, with autonomous-verifiable verdict)

Verdict legend: **AV** = autonomous-verifiable (pure porcelain parse + argv construction +
deterministic real-git-in-tempdir tests, the suaegi-git pattern). **AV/bare** = AV *only against a
local bare remote in a tempdir* (mechanics deterministic; live auth/network is not). **UI** =
inherently render-coupled. **NET** = needs live network + credentials. **DESTRUCTIVE** = can lose
committed/uncommitted work.

### 0.1 Log / history — **AV, highest clone value**
- **What:** the commit list the UI shows for a worktree — "what did the agent do here". Resolves
  HEAD, current ref (branch or detached), upstream ref, base ref, merge-base; runs
  `git log --format=<custom> -z --topo-order --decorate=full -n<limit+1> HEAD`; parses each record
  into `{id, parentIds, subject, message, author, authorEmail, timestamp, references[]}`; derives
  `hasIncomingChanges`/`hasOutgoingChanges` from merge-base vs upstream; `hasMore` from the `+1`.
- **Orca:** `main/git/history.ts:7` `getHistory` → `shared/git-history.ts:158`
  `loadGitHistoryFromExecutor` (resolve/clamp/merge-base orchestration) → `shared/git-history-log-parser.ts`
  (`GIT_HISTORY_COMMIT_FORMAT`, `parseGitHistoryLog`, `parseGitDecorationRefs`). Types in
  `shared/git-history-types.ts` (`GIT_HISTORY_DEFAULT_LIMIT=50`, `MAX=200`).
- **Verdict: AV.** Pure argv + porcelain parse; deterministic in tempdir (make N commits/branches/
  tags, assert list + refs + hasMore). The **graph swimlanes** (`shared/git-history-graph.ts` —
  lane/color assignment view-model) are pure but exist only to render the graph column → **defer
  with the UI (UI)**. Ship the flat commit list + decorations now.

### 0.2 Commit inspection / commit-show — **AV, high clone value**
- **What:** show a single commit's diff (the agent's commit, file-by-file). Resolve `commitId` →
  OID (`rev-parse --verify --end-of-options <id>^{commit}`); find first parent
  (`rev-list --parents -n1`); diff `parent..commit` via `diff --name-status -M -C` + `--numstat`
  (root commit → `diff-tree --root --no-commit-id` against the empty tree, no hardcoded hash);
  per-file content via blob reads at OIDs (`commitDiffEntry` → `readBlobAtOid`).
- **Orca:** `relay/git-handler-commit-diff-ops.ts:15` `commitCompare`, `:124` `commitDiffEntry`;
  `main/git/commit-object-ref.ts` (`hasLocalCommitObject`, `isFullGitObjectId`, the
  `^{commit}` verify). IPC surface: `relay/git-handler.ts:214` `git.commitCompare`.
- **Verdict: AV, strong.** Reuses the *exact* name-status/numstat parsing suaegi already owns in
  `compare.rs`. Deterministic in tempdir. Full-object-id assertion (reject non-40/64-hex) is the
  argv-injection guard — a pure unit test.

### 0.3 Branch read (list + current) — **AV, medium value**
- **What:** local branch short-names + which is current, for pickers. `for-each-ref
  --format=%(HEAD)%09%(refname:short) refs/heads/`, current-first sort. Scriptable, locale-stable
  (deliberately not `git branch`).
- **Orca:** `main/git/checkout.ts:41` `listLocalBranches`.
- **Verdict: AV.** Trivial pure parse, deterministic.

### 0.4 Branch switch / checkout — **AV, but mutating; lower value in the worktree model**
- **What:** `git checkout <branch> --` with `assertValidBranchName` (reject empty / leading `-`);
  trailing `--` disambiguates ref-vs-pathspec; git self-refuses on a dirty conflict and the error
  propagates (no `--force`).
- **Orca:** `main/git/checkout.ts:25` `checkoutBranch`, `:11` `assertValidBranchName`.
- **Verdict: AV, deterministic in tempdir.** Mutates worktree HEAD but is **not data-loss** (git
  refuses when a checkout would overwrite). Lower clone value: in the coding-agent-on-worktrees
  model each worktree is already pinned to its own branch, so in-place switching is rare. Include
  as an optional tail of the branch milestone or defer with branch-write.

### 0.5 Branch rename — **AV core, niche value**
- **What:** `git branch -m <new>`, gated by an upstream probe that *refuses* to rename a branch with
  an upstream (would orphan the remote branch / break an open PR), plus unique-name suffixing.
- **Orca:** `main/git/branch-rename.ts:81` `renameCurrentBranch`, `probeBranchUpstream`,
  `resolveUniqueBranchName`.
- **Verdict: AV for local mechanics** (rename + collision suffixing + `show-ref` existence). The
  upstream guard reads config (still local, AV). Niche — Orca uses it only for auto-rename-before-
  first-push. **Defer** with the remote milestone that gives it a reason to exist.

### 0.6 Branch create / standalone delete — **partially AV, DESTRUCTIVE (delete)**
- **What:** Orca has no first-class "create branch" op — branches are born from *worktree creation*
  (`worktree.ts` `add -b`). Deletion is fused to worktree cleanup + squash-merge safety
  (`update-ref -d <ref> <expected-head>`, "merged on any target" proof, checked-out-elsewhere
  guard) in `relay/git-handler-branch-cleanup.ts`.
- **Verdict:** create is AV but low-value (worktree create already covers it). Standalone delete is
  **DESTRUCTIVE** and, in Orca, inseparable from worktree lifecycle — suaegi already deletes
  branches inside `worktree.rs`. **Defer**; no clean standalone target.

### 0.7 Upstream ahead/behind status — **AV (local-only), medium value**
- **What:** `hasUpstream` + ahead/behind counts + patch-equivalence (`log --cherry-mark
  --right-only HEAD...@{upstream}`). Reads local refs/remote-tracking refs — **no network**.
- **Orca:** `main/git/upstream.ts` `getUpstreamStatus` → `shared/git-effective-upstream.ts`.
- **Verdict: AV, self-contained** (set upstream + commit both sides in a tempdir bare-remote pair,
  assert counts). Note: `git-history.ts` already computes `hasIncomingChanges`/`hasOutgoingChanges`
  via merge-base, so much of the *signal* rides along with §0.1 for free. A standalone numeric
  ahead/behind op is a small optional add.

### 0.8 Push / pull / fetch — **AV/bare mechanics, NET+credentials live → DEFER**
- **What:** `gitPush` (`push [--force-with-lease] --set-upstream <remote> <refspec>`, configured-
  upstream vs first-publish), `gitPull`/`gitFastForward`/`gitPullRebaseFromBase` (divergent-pull
  merge fallback), `gitFetch` (`fetch --prune`). Push-target validation via `check-ref-format
  --branch`. Error handling: `normalizeGitErrorMessage` classifies non-fast-forward / fetch-first /
  push-hook / submodule failures and `stripCredentialsFromMessage` scrubs `user:pass@` / token URLs
  before the message crosses the IPC boundary.
- **Orca:** `main/git/remote.ts:179` `gitPush`, `:252` `gitPull`, `:275` `gitPullRebaseFromBase`,
  `:296` `gitFetch`; `main/git/push-target-validation.ts`; `shared/git-remote-error.ts`
  (`stripCredentialsFromMessage`, `normalizeGitErrorMessage`, `runPullWithDivergenceFallback`);
  `main/git/fetch-error-classification.ts`.
- **Verdict: split.** The *pure* parts — argv construction, `check-ref-format` validation,
  credential scrubbing, non-fast-forward/divergence classification, the merge fallback — are all
  AV and unit-testable, and the round-trip mechanics (fast-forward, non-ff rejection, divergence)
  are **AV/bare** against a local bare remote (no network, no auth). What is **not** autonomously
  verifiable: real HTTPS/SSH auth, credential helpers, host rate limits. That surface needs
  **suaegi-secrets** (Plan 7 credential handling) to exist first. **DEFER to its own secrets-gated
  milestone**; do not fold live remote sync into this read-layer plan.

### 0.9 Remote / ref URL inspection — **AV, small value**
- **What:** `getRemoteUrl`, `getDefaultRemote`, `getRemoteCount`, and web-URL builders
  `getRemoteFileUrl`/`getRemoteCommitUrl` (remote URL → github/gitlab web link for "open in
  browser").
- **Orca:** `main/git/repo.ts:421,542,756,990,1010`.
- **Verdict: AV** (config reads + URL parsing, deterministic). Low standalone value; pairs
  naturally with a future "open in host" UI. **Defer**, optional.

### 0.10 Not present in Orca (no ground truth) — **skip / defer**
- **Stash** — *not implemented in Orca git* (grep: `stash` appears only in remote-error strings and
  unrelated modules). No porcelain to port. **Skip** unless suaegi wants a green-field feature.
- **Blame** — *not in Orca git* (only `shared/skill-freshness.ts` + a renderer github slice).
  Editor-adjacent. **Defer with the editor milestone.**
- **cherry-pick / revert / reset** — no user-facing operations in Orca git; they surface only as
  *conflict-state detection*, which `suaegi-git/conflict.rs` already models. **Defer**; would be
  DESTRUCTIVE green-field work with no clone value.

## §1 — Recommended milestone set + ordering

Ship a coherent **"git-depth read layer"**: the read-mostly, fully-autonomous, self-contained
operations the coding-agent-on-worktrees UI most needs — *what the agent committed*, and *show me
each commit*. Everything mutating, networked, or without Orca ground truth is deferred.

**NOW — smallest-first, each independently landable:**

- **M1 — Log / history** (`§0.1`). The keystone: the commit list per worktree. New module
  `history.rs`. Flat list + decorations + hasIncoming/hasOutgoing/hasMore. **Defer the graph
  swimlanes** to the UI plan.
- **M2 — Commit-show** (`§0.2`). Single-commit diff, file-by-file, reusing `compare.rs`'s
  name-status/numstat parsers and `file_diff` blob machinery. Depends conceptually on M1 (the
  history list is where a user clicks a commit) but is code-independent.
- **M3 — Branch read** (`§0.3`). List local branches + current, for pickers. Tiny, pure. Optional
  tail **M3b — branch checkout** (`§0.4`) if a low-risk mutating op is wanted; otherwise defer it.

Optional small add if scope allows: **ahead/behind numeric status** (`§0.7`) — but note M1 already
carries the incoming/outgoing *signal*, so this is marginal.

**DEFERRED (with reason):**

| Operation | Why deferred |
|---|---|
| push / pull / fetch (§0.8) | Live network + credentials → needs **suaegi-secrets**; own milestone. Mechanics are AV/bare but the auth surface is not autonomously verifiable. |
| branch rename (§0.5) | Niche; exists in Orca only to serve auto-rename-before-push. Ship with the remote milestone. |
| branch create / standalone delete (§0.6) | Create is subsumed by worktree-create; delete is DESTRUCTIVE and fused to worktree lifecycle (already partly in `worktree.rs`). |
| history graph swimlanes (§0.1) | Pure but render-only; belongs to the source-control UI plan. |
| upstream ahead/behind standalone (§0.7) | Signal already rides M1; promote only if the UI needs exact counts. |
| remote/ref URL builders (§0.9) | Low value alone; pair with an "open in host" UI. |
| stash / blame / cherry-pick / revert / reset (§0.10) | No Orca ground truth (stash/blame) or no user-facing op + DESTRUCTIVE (cherry-pick/revert/reset). |

Ordering rationale: M1 and M2 are the two highest clone-value operations *and* both are fully
autonomous *and* both reuse machinery suaegi already owns — maximal value at minimal risk. M3 is
trivial support. Nothing in the NOW set touches the network, writes user config, or risks data loss.

## §2 — Per-milestone crux / risk

### M1 — history
- **Custom format + dual separators.** `%H%n%aN%n%aE%n%at%n%ct%n%P%n%(decorate…sep=%x1f)%n%B` with
  `-z` between *records*. The decoration field uses `0x1f` as its internal separator because ref
  names may contain commas (`git-history-log-parser.ts` comment). Body `%B` is multi-line → it is
  `lines[7..].join('\n')` after the fixed header lines; subject is the trimmed first body line.
- **Decoration parsing.** Handle `HEAD -> refs/heads/x`, bare `refs/heads/x`, `refs/remotes/o/x`,
  `tag: refs/tags/x`; drop `HEAD` and `refs/remotes/*/HEAD`. Categorize + sort (branches, remote,
  tags). Port `parseGitDecorationRefs` + `compareGitHistoryItemRefsByCategory`.
- **Unborn / detached HEAD.** No HEAD commit → return an **empty** history, not an error (Orca:
  `loadGitHistoryFromExecutor` early-returns when `resolveCommit('HEAD')` is null). Detached HEAD →
  `currentRef` is the short hash, `category: 'commits'`. This is the *transient≠false-negative*
  invariant applied to reads.
- **hasMore via `-n limit+1`** then slice to `limit`. Clamp limit to [1, 200].
- **Hash width.** Orca's hash regex accepts **40 or 64 hex** (SHA-256 repos). Decide whether suaegi
  supports SHA-256 now (§4).
- **Encoding.** `%aN`/`%aE`/`%B` can be non-UTF-8. `GitRunner.run` returns `String` (lossy); author
  names with non-UTF-8 bytes would be mangled. Decide `run` (lossy, simple) vs `run_bytes` +
  explicit decode (§4). No data-loss risk either way — display only.
- **Data-loss:** none (pure read). **User-global-config:** never touched (all reads via GitRunner,
  which injects no identity/`-c`).

### M2 — commit-show
- **Root commit.** No parent → `diff-tree --root --no-commit-id --name-status/--numstat -r -M -C`
  (never hardcode the empty-tree hash — `--root` lets git pick the right one for SHA-1/256).
- **First-parent selection** via `rev-list --parents -n1` (parse first parent OID). Merge commits:
  Orca diffs against the *first* parent only.
- **Argv-injection guard.** Assert `commitId`/`parentOid` are full 40/64-hex before use — this is
  what stops a `-`-leading or ref-expression argument. Pure unit test; must be mutation-verified.
- **numstat rename form — reuse `-z`, do NOT port brace-parsing.** Orca's `commitCompare` uses
  *text* `--numstat` and therefore needs `normalizeNumstatPath` to unbrace `dir/{old => new}/f` and
  `old => new` (`shared/git-uncommitted-line-stats.ts`). suaegi's `compare.rs` already parses **`-z`
  numstat** (two-NUL-record rename form) cleanly. **Recommendation: use `-z` for commit-show's
  numstat too** and route through the existing parser, avoiding the brace-normalization path
  entirely. Same for name-status (`-z`, `core.quotePath=false`) — already handled by `compare.rs`.
- **Blob reads** at OIDs for per-file diff → reuse `compare.rs` `file_diff`/`file_head_bytes` blob
  path (binary detection already there).
- **Data-loss:** none. **Layering:** shares `ChangedFile`/`ChangeStatus`/numstat+name-status
  parsers with `compare.rs` (§3).

### M3 — branch read (+ optional M3b checkout)
- **Read** is pure (`for-each-ref`, tab-split, current-first sort). Zero risk.
- **Checkout (if included):** `assertValidBranchName` (reject empty / leading `-`) + trailing `--`
  are the injection/ambiguity guards. Mutating but **not data-loss** — git refuses a conflicting
  checkout and the raw message must propagate (Orca deliberately does not `--force`). Deterministic
  in tempdir (make branches, checkout, assert HEAD; assert dirty-conflict refusal).

### Deferred — push/pull/fetch (documented here so the later plan inherits the risk list)
- Network + credentials: **DEFER until suaegi-secrets exists.** When ported: scrub credentials from
  every error before it leaves the process (`stripCredentialsFromMessage` — `user:pass@` any scheme,
  lone `user@` only on http/s so SSH `git@host` survives); classify non-fast-forward / fetch-first;
  divergent-pull merge fallback (`--no-rebase`) only when the user pinned no strategy;
  `check-ref-format --branch` on any push target; `GIT_TERMINAL_PROMPT=0` (already in GitRunner)
  prevents credential-prompt hangs. Testable against a local bare remote; the live-auth surface is
  not autonomously verifiable — that gap must be stated explicitly in that plan.

## §3 — Landing module / layering / invariants

**Modules & layering (all run through `GitRunner` — inherit timeout/output-cap/tree-kill/
`GIT_TERMINAL_PROMPT=0`; raw `Command` forbidden; never inject identity or `-c` config):**

- **M1 → new `history.rs`** in `suaegi-git`. Owns the custom-format argv, `-z` record split,
  header/body slicing, decoration parser + category sort, and the resolve/merge-base orchestration
  (current/upstream/base refs, hasIncoming/hasOutgoing/hasMore, unborn/detached handling). Depends
  only on `runner.rs`. New public types mirror `GitHistoryItem`/`GitHistoryItemRef`/`GitHistoryResult`.
- **M2 → `compare.rs` (extend) or sibling `commit_show.rs` that calls `compare.rs`'s parsers.**
  It *must* reuse `ChangedFile`/`ChangeStatus`/the `-z` name-status + `-z` numstat parsers +
  `file_diff` blob path — do not duplicate them. Adds only: commit-OID resolution/verification,
  first-parent lookup, root-commit `diff-tree --root` branch, and the full-hex argv guard. Prefer a
  thin `commit_show.rs` that imports the shared parsers to keep `compare.rs` focused, unless the
  parsers aren't currently `pub(crate)`-reachable (§4).
- **M3 → new `branch.rs`.** `list_local_branches` (+ optional `checkout_branch`,
  `assert_valid_branch_name`). Depends only on `runner.rs`.
- All exported from `lib.rs`. No new deps.

**Invariants (from repo memory + Orca):**
- **Never write user global config.** M1–M3 are reads (+ one optional local-HEAD checkout); none
  passes identity or `-c user.*`. GitRunner already guarantees no `-c` injection — keep it that way.
- **Transient ≠ false-negative.** Unborn/detached HEAD, an unresolved base ref, or a missing
  upstream are *expected states* → empty/None results, **not** errors (Orca's history early-returns
  and `try/catch`-to-`undefined` throughout). A genuine corruption/permission error must still
  surface. Do not swallow broadly.
- **Data-loss discipline** applies only if M3b checkout ships — and checkout is non-data-loss (git
  self-guards); still, no `--force`. Nothing else in the NOW set mutates. The DESTRUCTIVE ops
  (reset/branch-delete/stash-drop) that *would* need the `DiscardOutcome`-style
  "what-did-I-do"-reporting discipline are all deferred.
- **Every regression test mutation-verified.** Per repo history, five hollow tests have already
  slipped through here. Each M1–M3 test must be shown to fail against a deliberately mutated
  implementation (wrong separator, wrong parent, dropped decoration, un-clamped limit, missing
  injection guard) before it counts.

## §4 — Open questions for Codex cross-validation

1. **Encoding — bytes vs lossy.** History author names / commit bodies (`%aN`/`%aE`/`%B`) and
   commit-show paths can be non-UTF-8. Use `GitRunner.run` (lossy `String`, simple) or `run_bytes`
   + explicit decode for the fields that can carry arbitrary bytes? Orca reads UTF-8 throughout.
2. **SHA-256 support now?** Orca's hash regex accepts 40 **or** 64 hex. Do we support SHA-256 repos
   in M1/M2 (regex + `--root` empty-tree already hash-agnostic), or assume SHA-1 and note it?
3. **commit-show home & parser reuse.** Extend `compare.rs` or add `commit_show.rs`? Are the `-z`
   name-status/numstat parsers and `file_diff` blob path currently reachable (`pub(crate)`) for a
   sibling module, or do they need light refactoring to share?
4. **numstat form.** Confirm the recommendation to use **`-z` numstat** for commit-show (reusing
   `compare.rs`) rather than porting Orca's text-form `normalizeNumstatPath` brace-unwrapping.
5. **Branch checkout — now or defer?** Include M3b (mutating but non-data-loss, AV) in the read
   layer, or defer it with the branch-write ops? Weigh against the worktree model where in-place
   switching is rare.
6. **Ahead/behind counts.** Is M1's `hasIncoming/hasOutgoing` signal enough, or does the UI need the
   exact numeric ahead/behind (`§0.7`) as a standalone op now?
7. **push/pull/fetch gating.** Confirm deferring *all* live remote sync until suaegi-secrets exists,
   and that the AV/bare mechanics (local-bare-remote tests) belong in *that* later plan, not this
   one. Any argument for landing the pure error-classification/credential-scrub helpers early?
8. **History graph swimlanes.** Confirm `git-history-graph.ts` (pure but render-only) is correctly
   deferred to the source-control UI plan rather than the backend read layer.
